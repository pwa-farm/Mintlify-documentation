Раздел **«PUSH-уведомления»** предназначен для создания, управления и отслеживания push-уведомлений, отправляемых пользователям PWA-приложения.

В верхней части страницы отображаются вкладки с количеством уведомлений по их статусу:

- **Все** — список всех созданных push-уведомлений.
- **Действующие** — активные уведомления, которые доступны для отправки или уже участвуют в рассылке.
- **Отключенные** — уведомления, которые были деактивированы и не отправляются пользователям.

Для быстрого поиска нужного уведомления предусмотрено **поле поиска** по названию.

В таблице отображается информация о каждом push-уведомлении:

- **Название** — название уведомления.
- **Дата создания** — дата, когда уведомление было создано.
- **Статус** — текущее состояние уведомления (например, **Активен** или **Отключен**).

Для каждого уведомления доступно **меню действий**, позволяющее выполнять операции с выбранным уведомлением (например, редактирование, отключение или удаление).

# **PUSH по событиям**

Раздел **«Создание PUSH по событиям»** предназначен для настройки автоматической отправки push-уведомлений пользователям при наступлении определенного события. После создания и активации такого уведомления система самостоятельно отправляет его новым пользователям, соответствующим заданным условиям.

При создании push-уведомления необходимо заполнить следующие параметры:

- **Системное название** — внутреннее название уведомления, которое отображается только в интерфейсе платформы и используется для удобства управления созданными push-уведомлениями.
- **Активировать рассылку** — переключатель, который включает или отключает автоматическую отправку уведомления. Если функция активирована, уведомление будет отправляться пользователям при выполнении выбранного события.
- **Условия отправки** — выбор события, которое служит триггером для отправки push-уведомления. Например, отправка может происходить после регистрации пользователя или другого доступного события. После наступления выбранного события запускается отсчет времени до отправки уведомления.
- **Timeout для отправки** — настройка задержки между наступлением события и отправкой уведомления. Задержка задается отдельно в:
  - днях;
  - часах;
  - минутах;
  - секундах.
  Это позволяет отправлять уведомление как сразу после события, так и спустя любой необходимый промежуток времени.

### Контент

Раздел **«Контент»** предназначен для настройки содержимого push-уведомления, которое будет отображаться пользователю после выполнения выбранного события.

В блоке доступны следующие параметры:

- **Локализация** — выбор языка или локализации, для которой создается текст уведомления. Если приложение поддерживает несколько языков, для каждой локализации можно настроить собственные заголовок и текст push-уведомления.
- **Заголовок пуша** — краткий заголовок уведомления, который отображается в верхней части push-сообщения. Заголовок должен быть информативным и привлекать внимание пользователя.
- **Текст пуша** — основное содержание уведомления. Здесь указывается сообщение, которое увидит пользователь после получения push-уведомления. Текст рекомендуется делать кратким, понятным и содержащим призыв к действию, если это необходимо.

Для каждой выбранной локализации можно задать собственный заголовок и текст уведомления, благодаря чему пользователи будут получать сообщения на соответствующем языке. Это позволяет повысить удобство взаимодействия и эффективность push-рассылок.

<Frame>
  ![Image](/images/image-38.png)
</Frame>

В разделе **«Медиа уведомления»** нужно настроить визуальное оформление push-уведомления.

Доступны три типа изображений:

- **Иконка** — основная иконка, отображаемая в push-уведомлении.
- **Логотип** — логотип уведомления. Он автоматически обрезается до соотношения сторон **1:1**.
- **Баннер** — широкое изображение для отображения в уведомлении.

<Frame>
  ![Image](/images/image-39.png)
</Frame>

В разделе **URL** нужно указать целевой адрес, который будет открыт при нажатии на push-уведомление.

- В поле **URL** вводится ссылка на страницу, которую должен открыть пользователь после нажатия на уведомление.
- Если поле оставить пустым, будет использована **оффер-ссылка**, указанная в конфигурации PWA-приложения.

### Настройка получателей

В разделе **«Получатели»** настраивается целевая аудитория для отправки push-уведомления.

Доступны следующие параметры:

- **PWA для отправки** — выбор одного или нескольких PWA-приложений, пользователям которых будет отправлено уведомление.
- **Фильтр по регистрациям**:
  - Отправлять всем;
  - Только пользователям, у которых была регистрация;
  - Только пользователям, у которых не было регистрации.
- **Фильтр по депозитам**:
  - Отправлять всем;
  - Только пользователям, у которых был депозит;
  - Только пользователям, у которых не было депозита.

Настройка этих параметров позволяет ограничить рассылку определенной аудиторией и отправить push-уведомление только пользователям, соответствующим выбранным условиям.

  ![Image](/images/image-41.png)
</Frame>

Для сохранения PUSH - уведомления нажимаем кнопку **Сохранить** в правом нижнем углу.

# **PUSH по календарю**

### Конфигурация PUSH по расписанию

В разделе **«Конфигурация пуша»** задаются основные параметры рассылки по расписанию.

Доступны следующие настройки:

- **Системное название** — внутреннее название рассылки, которое отображается в интерфейсе [pwa.farm](http://pwa.farm) и используется для ее идентификации. Пользователи не видят это название.
- **Активировать рассылку** — переключатель, позволяющий включить или отключить отправку push-уведомлений. После активации уведомления будут отправляться в соответствии с заданным расписанием.
- **Цвет события** — выбор цветовой метки для визуальной идентификации рассылки в календаре PWA Farm. Цвет используется только для отображения в интерфейсе и не влияет на отправку или содержимое push-уведомления.
  <Frame>
    ![Image](/images/image-42.png)
  </Frame>

### Расписание

В разделе **«Расписание»** настраиваются параметры отправки push-уведомления по времени.

Доступны следующие настройки:

- **Часовой пояс для отправки** — выбор часового пояса, относительно которого будет выполняться отправка push-уведомлений. Это позволяет отправлять уведомления в заданное локальное время независимо от местоположения пользователей.
- **Тип расписания** — определяет периодичность отправки уведомлений:
  - **Однократно** — уведомление будет отправлено один раз в указанную дату и время.
  - **Регулярно** — уведомление будет отправляться по повторяющемуся расписанию (после выбора этого режима становятся доступны дополнительные параметры настройки периодичности).
- **Дата и время отправки** — поле для выбора даты и времени первой или единственной отправки уведомления в соответствии с выбранным типом расписания. Дата и время указываются с учетом выбранного часового пояса.

- **Заголовок пуша** — ввод заголовка уведомления, который отображается пользователю в списке уведомлений и в самом push-сообщении.
